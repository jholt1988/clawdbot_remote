## Pre-Lease Setup
- [ ] Confirm latest template version is selected (`PDL-Blank LEase-1.pdf`)
- [ ] Confirm required clauses for current jurisdiction are present
- [ ] Confirm tenant screening criteria are current (`Tenant-Screening.md`)
- [ ] Confirm lease lifecycle process is aligned (`lease-lifecycle-upgrade-plan.md`)

## Applicant Screening
- [ ] Identity verification complete
- [ ] Income/employment verification complete
- [ ] Rental history/references checked
- [ ] Screening result documented against criteria
- [ ] Approval/denial rationale recorded

## Lease Preparation
- [ ] Tenant legal name and property/unit details validated
- [ ] Rent amount, due date, fees, deposit, term dates validated
- [ ] Maintenance obligations and communication channels included
- [ ] Required disclosures attached

## Signing & Activation
- [ ] Signature workflow completed (all parties)
- [ ] Executed lease stored in canonical location
- [ ] Tenant portal docs visibility confirmed
- [ ] Billing/autopay setup confirmed

## Post-Signing Controls
- [ ] Move-in checklist triggered
- [ ] Reminder schedule set (renewal, inspections, notices)
- [ ] Initial maintenance contact protocol sent
- [ ] Owner/PM visibility and permissions verified

## Renewal / Change / Exit
- [ ] Renewal timeline initiated on schedule
- [ ] Any amendments captured with audit trail
- [ ] Move-out process + deposit workflow initialized when applicable

---

## Quality Gate (Definition of Done)
A lease is “ops-complete” only when:
1. Screening decision is documented,
2. Signed lease is stored + accessible,
3. Billing + communication are active,
4. Lifecycle reminders are scheduled.
