## 1) Pre-work gate
- [ ] Identify project.
- [ ] Identify truth domain.
- [ ] Identify canonical source.
- [ ] Read canonical.

## 2) Update gate
- [ ] Write canonical first.
- [ ] Update Last Verified + Change Log (if present).
- [ ] Update derivatives after.
- [ ] Ensure backlinks.

## 3) Cadence gate
Daily:
- [ ] Tickets reflect reality.
- [ ] Calendar reflects intent.
- [ ] Drift captures interruptions/decisions.

Weekly:
- [ ] Review milestones.
- [ ] Prune stale tickets.
- [ ] Identify recurring drift causes.

## 4) Shipping gate
- [ ] Acceptance criteria recorded.
- [ ] Artifact link/path recorded.
- [ ] Status updated.
- [ ] Downstream systems reconciled.

## 5) Closure gate
- [ ] Mark canonical status.
- [ ] Write debrief.
- [ ] Archive/demote drift-causing derivatives.
