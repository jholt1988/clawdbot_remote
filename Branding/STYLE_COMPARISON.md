# PMS Style Comparison: Current vs Alternate

## Summary
This comparison maps the existing PMS style to the proposed **Modern Ledger** alternate direction so teams can choose by context instead of preference.

## Side-by-side

| Dimension | Current Style (from existing guide) | Alternate Style (Modern Ledger) | Best fit |
|---|---|---|---|
| Brand expression | Bright modern SaaS utility | Calm premium operational clarity | Choose based on audience seniority |
| Core palette | Navy + electric blue + teal | Trust navy + structured neutrals + restrained amber | Alternate for enterprise confidence |
| Accent intensity | Higher saturation | Lower saturation, more neutral field | Alternate for data-heavy and formal contexts |
| Typography | Inter-forward | Manrope (headlines) + Inter (body) | Alternate for editorial/pitch polish |
| Components | Clean cards, restrained shadows | Border-led structure, subtle depth, tighter hierarchy | Alternate for dense workflows |
| Imagery | General modern SaaS style | Real operations + architecture, lightly desaturated | Alternate for credibility and realism |
| Iconography | Practical icon usage | Strict outline system, semantic discipline | Alternate for consistency at scale |
| Motion | Not heavily specified | Brief, predictable, low-drama motion | Alternate for enterprise UI norms |
| Messaging | Clear and practical | Clear + evidence-led + executive-ready | Alternate for buyer-facing channels |

## Recommended use cases by surface

### 1) Enterprise sales / procurement
**Recommended:** Alternate (Modern Ledger)
- Why: signals stability, control, and operational maturity
- Assets: deck templates, one-pagers, security/compliance collateral

### 2) Marketing website
**Recommended:** Hybrid
- Hero + top nav: Alternate tone
- Product feature sections: Current or hybrid for clarity
- CTAs: keep strong contrast; avoid oversaturated accents

### 3) Product UI
**Recommended:** Current as default, Alternate as opt-in theme or enterprise skin
- Why: current style is already implementation-friendly
- Alternate can improve readability for admin-heavy workflows

### 4) Pitch decks (investor/partner)
**Recommended:** Alternate
- Why: stronger executive polish, cleaner hierarchy, less visual noise

### 5) Onboarding/help center docs
**Recommended:** Current with selective alternate neutrals
- Why: current style already optimized for directness; alternate neutrals can improve reading comfort

## Decision framework
Use **Current** when you need:
- Faster implementation continuity
- Higher visual energy
- Existing UI parity with minimal changes

Use **Alternate** when you need:
- Executive trust signaling
- Enterprise procurement readiness
- Dense data readability and calmer visual rhythm

## Migration strategy (practical)
1. Keep logo asset unchanged.
2. Introduce alternate tokens + typography in a separate theme.
3. Start with external collateral (decks/case studies), then marketing pages, then optional product theme.
4. Track key outcomes: conversion quality, time on page, perceived trust in sales calls.

## Assumptions
1. Current style baseline is defined by `Branding/BRAND_STYLE_GUIDE.md` and `Branding/BRAND_QUICK_REFERENCE.md`.
2. Modern logo concept pillars (security/simplicity/centrality) remain non-negotiable.
3. Alternate direction should coexist with, not replace, existing implementation unless explicitly approved.
