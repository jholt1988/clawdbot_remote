# Brand Style Guide (PMS)

## 1) Brand foundation
- **Product name:** Property Management Suite (PMS)
- **Positioning:** Modern, trustworthy operating system for property operations.
- **Voice:** Clear, practical, confident, low-jargon.
- **Tone:** Professional by default; supportive in onboarding/help contexts.

## 2) Logo system
Primary asset source: `Branding/Modern_Logo_1.png`

### Approved usage
- Use logo on clean, high-contrast backgrounds.
- Preserve aspect ratio; never stretch/skew.
- Maintain clear space at least equal to logo icon height on all sides.

### Minimum size
- Digital min width: **120 px**
- Print min width: **25 mm**

### Misuse (do not)
- Recolor logo arbitrarily
- Add outlines/shadows/glows
- Rotate or distort
- Place over busy/low-contrast imagery

## 3) Color system (recommended)
> Based on existing PMS visual direction; adjust to final design tokens if needed.

- **Primary:** Deep Navy `#0B1220`
- **Primary Accent:** Electric Blue `#2F80FF`
- **Secondary Accent:** Teal `#13C2C2`
- **Success:** `#10B981`
- **Warning:** `#F59E0B`
- **Danger:** `#EF4444`
- **Neutral 900/700/500/200/50:** `#111827 / #374151 / #6B7280 / #E5E7EB / #F9FAFB`

### Accessibility
- Body text should maintain **WCAG AA (4.5:1)** contrast.
- Buttons/interactive labels should maintain **AA for UI components**.

## 4) Typography
- **Heading font:** Inter / system sans fallback
- **Body font:** Inter / system sans fallback
- **Monospace:** JetBrains Mono (code/IDs)

### Hierarchy
- H1: 32/40, 700
- H2: 24/32, 600
- H3: 20/28, 600
- Body: 16/24, 400
- Caption: 12/16, 400

## 5) UI expression
- Use clean cards, restrained shadows, clear spacing.
- Status colors: success/warning/danger reserved for state semantics only.
- Prefer explicit labels over ambiguous icon-only controls.

## 6) Messaging style
### Do
- Lead with outcome and action.
- Use direct CTAs: “Start Application”, “Approve”, “Request Info”.
- Keep compliance/legal copy plain and unambiguous.

### Don’t
- Use hype-heavy claims without proof.
- Mix product naming variants inconsistently.
- Overload screens with decorative text.

## 7) Channel usage quick rules
- **Docs:** Left-aligned, structured headings, procedural clarity first.
- **Slides:** One message per slide; strong contrast; minimal dense text.
- **Social snippets:** Short, benefit-first, link to canonical doc.

## 8) Governance
- Canonical brand assets live in `Branding/`.
- Any logo/color/type changes require changelog note and before/after samples.
